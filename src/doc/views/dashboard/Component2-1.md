# Component2 模型注册组件
该组件位于`src\views\dashboard\index.vue`中，
你完成`src\views\dashboard\components\Component2.vue`这个组件的渲染展示即可

## 需求
使用vue+ts完成页面组件
该组件用于展示后端已经添加的模型分组和其对应的各种模型，你暂时不需要写后端逻辑，假数据即可
各种模型对应的是一个分组(提供商)的各种模型，比如openai的gpt-5.4和image2分别为不同类别的模型

分组渲染时，你需要渲染一个向下的箭头，让用户选择对应的分组(提供商)
选择后，会请求后端获取对应分组的各类模型，有多少类模型，你就渲染多少个块，各个块内，顶部是模型类别，

## 页面描述
整个分页组件背景为rgba(249, 249, 249, 0.85)，和上部topbar和左边nav有一定间隙不要贴紧，里面有1+n(提供商下的模型分类数量)

1.顶部：左边为一大块用于展示提供商信息右边留一个小块用于添加模型(你先做一个块渲染即可，点击弹出的新增弹框不用做)
位置：提供商信息展示块，左边为提供商的图片居中，右边从上至下依次渲染，提供商名称，默认api-key，模型总数，创建时间

2.中部：渲染该提供商下注册的各个分类的模型，一行渲染三类，一类一个块，居中平分位置，留一定空隙
每个块：上部是块分类名称：比如：文生图，聊天模型，嵌入模型等，下部则渲染两列模型名称，每列五个，不用理会超出范围的情况先(我后端会保证不返回超过10个该分类的模型)，然后鼠标hover到这个模型名称，要有变色动画，然后在后面还会出现一个小垃圾桶或者x，你先用叉叉吧，后面我找到对应svg放入，点击这个就可以删除这个分类下的模型。

3.底部就是一句小tips，你先随便写一点，一句话即可，靠左边，颜色要和组件背景色区分开来，最好可以一直贴在底部，中部渲染的分类较多时，组件出现下拉条，下拉时这句话消失，下拉完再从底部冒出来
